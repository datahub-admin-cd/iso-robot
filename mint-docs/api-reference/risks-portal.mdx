## Upload selected risks

```http
POST /risks/upload-selected
```

**Auth:** Bearer

Persists the analyst's chosen risks to the organisation's `risks` table.

### Request

```json
{
  "client_org_id": "uuid",
  "selected_risks": [
    {
      "issue_id": "uuid",
      "risk_title": "Quarterly access review gaps",
      "risk_rating": "Medium"
    }
  ]
}
```

### Response `200`

```json
{ "status": "success", "data": { "created": ["uuid"] } }
```

---

## List risks for org

```http
GET /risks/{orgId}
```

**Auth:** Bearer

Returns the organisation's risk register — each risk reflects its current
tagging state (`tag_status`, `*_tags`) and owner-assignment state
(`owner_assignment_status`, `owner`, `accountable`), in addition to the core
risk fields.

The `owner` and `accountable` objects are resolved from the organisation
hierarchy (`org_hierarchy_users`, by `owner_user_id` / `accountable_user_id`)
and are `null` until an owner/accountable user has been assigned.
